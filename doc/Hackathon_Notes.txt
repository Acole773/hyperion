FEEDBACK

Just a few random comments:

1). As you are already working on - the timers should be in the code to insulate the timings 
	from response time fluctuations on the login node.
2). I would recommend to test your outer job submission script with the 
	https://code.ornl.gov/olcf/hello_jobstep utility to ensure that the CPU-core and 
	GPU-affinity/placement is what you want.
3). Regarding the SLURM batch script clause "#SBATCH --array=0-4", I am not certain if it's guaranteed 
	that different job steps never run concurrently on the same compute node.  If they were to run 
	concurrently on the same node then that could also contaminate timings due to some shared 
	resources on the compute node, i.e., file-I/O  and memory CPU memory bandwidth.
4). I assume that all input- and output-data are under /lustre/orion/* and not under $HOME and the jobs 
	are run from lustre.
